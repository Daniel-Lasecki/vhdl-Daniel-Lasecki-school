### https://git-scm.com/video/what-is-version-control (5:58) Q1: What are the benefits of using git version control?

* Git provides independent developers or teams with an easy way to document, save, review, and revert changes as needed. For teams, it offers a tool that allows multiple people to work on the same files simultaneously, minimizing conflicts atleast most of the time.

### https://git-scm.com/video/what-is-git (8:15) Q2: What is a distributed version control system?
* Distributed version control system means that every member of the project has a complete copy of the project repository, including its full history, on their local machine. This allows them to work independently in their own "timeline," making changes and commits locally. Once they are ready, they can sync their changes with others by pushing their updates to a main/master branch, where everyoneâ€™s work is combined.

### https://git-scm.com/video/get-going (4:26) Q3: What are the most important git commands in terminal use?
* git init xxx  (Creates git project folder)
* git add xxx.file (Adds specific files to local commit box not publishing yet to upstream)
* git commit -m "message why commit is made" (Commits every file from local commit box)

### Introduction to GitLab (59:51) Q4: Explain "GitLab Recommended Process" Q5: What is a "Merge Request"? (Video is not aviable)

Q4: Gitlabs recomennded process is first to clone or fork main repository to create their own local version of the project. After which they should create local branch for each new feature so it can be tracked easily and reviewed or make changes. And when developer is ready they should push changes to gitlab to make a cloud save of their version. If everything is ready to be published to central project it will be merged together.

Q5: Merge request is a feature to propose changes from local branch into main branch. It allows team members to review code, propose changes and add comments.
